saskavi

run code, not servers

what is saskavi?

saskavi frees you from the cognitive burden of writing web-apps.

Instead of writing web-servers which respond to web-requests, you deploy modules written in your favorite language. These modules are then made available to your client side scripts over a fast and secure communication platform.

At saskavi, we like to call it Isolated Auto-scaled Trusted Code Execution.

features

Write functions in your favorite language instead of writing web-request handlers

Run from anywhere. Locally on your development machine, or on the saskavi scaled infrastructure.

usage

Your trusted code

var paymentGateway = createGateway("SECRET_STUFF");

module.exports.processPayment = function(customerId, amount, cb) {
    paymentGateway.processPayment(customerId,
        amount, function(err, res) {
        log("Payment processing result:", err, res);
        cb(err, res);
    }
};
					
module TrustedCode
	def self.processPayment(customerId, amount)
		return paymentGateway.processPayment(
			customerId,
			amount);
	end
end


					
def processPayment(customerId, amount)
	return paymentGateway.processPayment(customerId, amount);

Your client side

var saskavi = new Saskavi("APP-ID");
saskavi.call("processPayment",
             "customer-id", 499, function(err, res) {
    if (err) return failed("Could not process payment");
    success("Payment was successfully processed");
});

Run locally


$ npm install -g saskavi
$ saskavi init
$ saskavi run

Or deploy to our infrastructure


$ npm install -g saskavi
$ saskavi init
$ saskavi deploy
